# TarlaBazar — прямой рынок от поля до покупателя

Второй сайт проекта. Халяльный B2B-маркетплейс: фермеры продают свой реальный,
уже собранный урожай напрямую магазинам, ресторанам и переработчикам — без
цепочки перекупщиков, которая поднимает цену в 2–3 раза.

Интерфейс на трёх языках: русский, английский, азербайджанский.
Палитра та же природная, что у AgroLens, — два сайта читаются как одна система.

## Почему это халяль — пять решений в самой конструкции

Халяльность здесь не наклейка, а устройство платформы. Каждый принцип убирает
конкретный запрещённый элемент обычного маркетплейса:

1. **Продаётся только реальный товар.** Фермер продаёт урожай, который у него
   физически есть и уже собран. Платформа никогда не владеет товаром и не
   перепродаёт его — нет дропшиппинга, нет продажи того, чего у продавца нет.
2. **Доход — комиссия за услугу, не процент.** Платформа берёт 2,5% за
   посредничество (симсара), дозволенное учёными. Ни займов под процент, ни
   рассрочки с наценкой — рибы нет.
3. **Каждый лот — с ясными условиями.** Культура, объём, цена за кг, сорт —
   указаны прямо. Это убирает гарар, чрезмерную неопределённость сделки.
4. **Только собранный урожай.** Нельзя продать будущий урожай, которого ещё нет
   на земле, — отдельный шариатский запрет. Никаких фьючерсов на несобранное.
5. **Деньги как честный залог.** Оплата замораживается у банка-партнёра как
   доверенный залог (амана) до получения товара и переводится фермеру без
   приращения. Платформа не держит чужие деньги и не начисляет процент. Это
   одновременно халяльно и законно — не требует лицензии на собственный эскроу.

Приятное совпадение: то, что делает сделку халяльной, делает её и устойчивой.
Международные агро-маркетплейсы разорялись именно на том, что запрещает шариат, —
на займах под процент и на владении товаром и складами.

## Что это за сайт технически

Рабочий каркас маркетплейса, а не бутафория. Размещённые лоты и отправленные
заявки реально сохраняются (в браузере, через localStorage) и живут между
сессиями. Платформа стартует пустой — никаких выдуманных фермеров и фейковых
сделок. Пустые состояния честно говорят, что данные появятся с приходом реальных
пользователей.

### Экраны

- **Главная** — hero с прямой связью фермер-покупатель, проблема цепочки
  посредников, три шага работы, пять принципов халяльности.
- **Каталог** (`/catalog`) — сетка лотов с фильтрами по культуре, региону, сорту.
  Пустое состояние с приглашением разместить первый лот.
- **Разместить лот** (`/new`) — форма с обязательной галкой «урожай собран»
  (халяльное условие зашито в код: продать несобранное нельзя).
- **Лот** (`/listing/:id`) — детали + форма заявки покупателя с живым расчётом
  суммы и комиссии, объяснением халяльного эскроу.
- **Заявки** (`/deals`) — жизненный цикл сделки: отправлена → принята →
  оплачена (в залоге у банка) → товар получен (оплата у фермера). Кнопки
  продвигают статус, полоса показывает поток.

## Как запустить

Нужен Node.js 18+.

```bash
npm install     # поставить зависимости
npm run dev     # локально → http://localhost:5173
npm run build   # собрать в папку dist/ для хостинга
npm run preview # посмотреть собранное
```

## Что нужно для боевого запуска (сверх этого каркаса)

Каркас показывает и проверяет всю логику. Для реального рынка нужны серверные
части, которые в браузере жить не могут:

- Общая база данных вместо localStorage — чтобы лоты одного фермера видели все.
- Интеграция с банком-партнёром (PASHA Bank, ABB и т.п.) по API для реального
  сплита платежа: покупатель платит на транзитный счёт, банк переводит фермеру,
  платформа получает комиссию. Своего эскроу-счёта платформа не ведёт — это
  требует лицензии платёжной организации и, при начислении процента, нарушало бы
  халяльность.
- Верификация фермера через ASAN Login по FIN/VÖEN.
- Стыковка с первым сайтом (AgroLens): сорт качества из фотоанализа
  подставляется в лот автоматически.
